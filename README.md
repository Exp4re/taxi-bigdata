# taxi-bigdata
# 基于分布式架构的纽约出租车数据分析课程设计
## 一、项目概述
本项目为大数据课程设计实战案例，采用**HDFS+YARN+Spark**三层分布式架构，以纽约TLC官方2024年1月黄色出租车运营数据为数据源，完成从数据分布式存储、数据预处理、分布式计算、结果持久化到可视化分析的全流程开发。项目充分利用分布式系统分块存储、并行计算的优势，解决海量订单数据单机处理效率低、存储受限的问题，从服务商营收、乘车人数、车费分布三个维度挖掘出租车出行规律。

## 二、技术架构
1. **存储层**：HDFS 分布式文件系统，存放原始数据、清洗数据、统计结果
2. **调度层**：YARN 资源调度框架，统一管理集群CPU与内存资源
3. **计算层**：Spark 分布式计算引擎，Scala语言编写业务代码
4. **可视化**：Python + Matplotlib 绘制分析图表

## 三、文件说明
- `TaxiAnalysis.scala`：Spark 数据读取、清洗、统计核心代码
- `plot_taxi.py`：Python 数据可视化代码
- `大数据编程课程设计论文.docx`：完整课程设计论文
- `images`：项目运行截图、架构图、可视化成果图

## 四、核心运行流程
1. 启动Hadoop、Spark伪分布式集群，通过`hdfs dfs`命令创建目录并上传原始数据；
2. 进入`spark-shell`执行代码，读取HDFS数据并过滤无效脏数据；
3. 分组聚合完成多维度统计，结果以CSV格式写回HDFS；
4. 下载统计文件至本地，使用Python生成可视化图表。

## 五、环境要求
操作系统：Ubuntu Linux
基础环境：JDK 1.8
大数据组件：Hadoop 3.4.2、Spark 3.x
编程语言：Scala、Python 3
